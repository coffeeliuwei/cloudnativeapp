# 实训指导书（学生版）—— 云原生应用：从本地到生产的完整上云实训

> **一句话目标**：用 24 课时，带你把一个**本地能跑的微服务项目（coffee 咖啡订单/快递系统）**，一步步搬上阿里云、接好自动化流水线、配齐网络与运维、做完安全加固，**最终达到可对外提供服务的生产级状态**。
>
> **主线理念**：**本地做过的每一步，云上一一对应替换**。不重学一套"云上的东西"，而是把本地的 MySQL / Nacos / 启动进程，逐个换成云上的 RDS / MSE Nacos / EDAS 托管——这就是"云原生"的本质。

## 实训一　环境搭建与项目认知（2 课时）

**实训目标**
- 能用一句话说清 coffee 项目的微服务调用链：浏览器 → `coffee-app`(网关) → Dubbo RPC → `coffee-userorder`/`coffee-expresstrack` → MySQL。
- 在本机装好并验证：JDK 17、MySQL 8.0、Maven 3.8+、Node.js LTS、Nacos 2.3.x。

**前置条件**：电脑能访问外网；建议提前配好国内镜像。

**实训任务**
1. 速读 [架构详解](02-architecture.md)、[各模块代码详解](03-modules.md)、[数据库设计](04-database.md)，画出（或填空）一张调用链路图。
2. 按 [快速启动指南](05-quick-start.md) 的"前置准备"逐项安装并执行验证命令：
   - JDK 17（Temurin LTS）→ `java -version` 显示 17
   - MySQL 8.0（Developer Default，端口记牢）→ `mysql -u root -p` 能进
   - Maven 3.8+（settings.xml 配阿里云镜像）→ `mvn -version` 的 Java 显示 17
   - Node.js **LTS**（18/20，**不要 Current/Node 24**）→ `node -v`
   - Nacos **2.3.x**（**不能用 3.x**），解压到无中文无空格路径

**预期效果 / 验收标准**
- 5 个 `-version` 验证命令全部通过且版本正确。
- 能复述调用链路，并说明"为什么 api 要单独成模块"（Dubbo consumer/provider 共享接口定义）。

**学生提交物**：5 张版本验证截图 + 一张自画调用链路图。

**常见问题**：JDK 不是 17 → 改 `JAVA_HOME`；Node 用了 24 → 装 LTS、必要时 `npm i -g npm@10`。

## 实训二　本地跑通微服务项目（4 课时）

**实训目标**：在本机把整套后端 + 前端按正确顺序启动并跑通完整业务链路，掌握常见错误的定位方法。

**前置条件**：实训一完成。

**实训任务**（严格按 [快速启动指南](05-quick-start.md) 阶段一）
1. **Step 1 启动 Nacos**：`startup.cmd -m standalone`，访问 `localhost:8848/nacos`（nacos/nacos）。**Nacos 必须最先启动**。
2. **Step 2 初始化数据库**：建 `userordertest`、`expresstracktest` 两个库，执行 `sql/` 下两个初始化脚本。
3. **Step 3 配置数据库连接**：改两个 `application-dev.yml` 的 `host` 端口（默认 3307，按实际 MySQL 端口改）、账号、密码。
4. **Step 4 安装本地依赖**：**先 `coffee-common`**，再两个 `api`，用 `mvn` 不用 `mvnw`。
5. **Step 5-7 启动服务**：顺序 `coffee-userorder`(7001) → `coffee-expresstrack`(8001/Dubbo 28888) → `coffee-app`(8005)，在 Nacos 服务列表确认注册。
6. **Step 8 测后端**：浏览器访问 `http://localhost:8005/hello/44556677` 返回 JSON。
7. **Step 9 启前端**：`app-admin` 里 `npm install` → `npm run dev`(8080)，登录后在"订单管理""轨迹查询"看到数据。

**预期效果 / 验收标准**
- Nacos 服务列表能看到 `coffee-userorder`、`coffee-expresstrack`。
- `/hello/44556677` 返回正确 JSON；前端轨迹查询 `44556677` 出现轨迹记录 → 整链路（前端→网关→Dubbo→provider→MySQL）跑通。

**学生提交物**：Nacos 服务列表截图、后端接口 JSON 截图、前端有数据截图。

**常见问题**（对照文档"常见错误速查"）：`10061` MySQL 没启动；`1251` 老客户端认证协议；`No provider available` 启动顺序；端口占用用 `netstat -ano | findstr :端口` + `taskkill`。

## 实训三　云资源准备（2 课时）

**实训目标**：在阿里云开好上云所需的基础资源，理解"同地域、同 VPC"这条铁律。

**前置条件**：阿里云账号已实名。

**实训任务**（[阿里云配置指南](01-aliyun-guide.md) + [EDAS 部署](06-edas-deployment.md) Part 2-3）
1. **RAM 子账号**：日常用子账号而非主账号，授必要权限（理念见实训九/安全）。
2. **VPC**：创建专有网络（如 `192.168.0.0/16`）+ 交换机；**记住地域和 VPC**。
3. **购买 3 台 ECS**：每台 **2 核 2 GB**、按量付费、**同地域同 VPC**，记下 3 台的公网/私网 IP（实训五部署要用：ECS-1 订单、ECS-2 快递、ECS-3 网关+前端）。
4. **RDS MySQL 8.0**：建实例 → 建 `userordertest`/`expresstracktest` 两库 → 建账号授权 → 白名单填 `172.16.0.0/16`（允许整个 VPC）→ 记内网/外网地址。
5. **MSE Nacos**：本项目 Dubbo 3.x **必须选专业版**（最少 3 节点）；网络选 ECS 同 VPC；记内网/公网地址；公网白名单临时 `0.0.0.0/0`（仅教学）。

> 🔥 **铁律**：ECS、RDS、MSE Nacos **必须同地域、同 VPC**，创建后大多不可改，错了只能删重建。

**预期效果 / 验收标准**
- 3 台 ECS、RDS、MSE 均"运行中"且同地域同 VPC；本机用外网地址能连上 RDS（灌表）、能连上 MSE（启动一个本地 provider 后在 MSE 服务列表看到它）。

**学生提交物**：3 台 ECS 列表、RDS 实例信息、MSE 实例信息、白名单设置截图（地址打码）。

**常见问题**：连不上多半是白名单没放行；地域/VPC 不一致导致内网不通。

## 实训四　平移上云：本地连云上（2 课时）

**实训目标**：**一行代码不改**，只改配置默认值/环境变量，让本地应用连上云上 RDS 与 MSE Nacos，理解"配置与代码分离"。

**前置条件**：实训二、三完成。

**实训任务**（[快速启动指南](05-quick-start.md) 阶段二）
1. **替换一**：把两个 `application-dev.yml` 的数据库 `host/user/password` 改成 **RDS 外网地址/账号/密码**，重启微服务，重测 `/hello/44556677`。
2. **替换二**：把 Dubbo 注册中心地址、Nacos 配置中心地址改成 **MSE 公网地址**，重启，在 MSE 控制台服务列表看到两个服务注册成功。
3. 理解 `${DB_HOST:默认值}` 写法：优先读环境变量，没设才用默认值——同一份 jar 可连任意环境。

**预期效果 / 验收标准**
- 本地应用查询的数据来自**云上 RDS**；服务注册在**云上 MSE**；接口仍返回正确 JSON。

**学生提交物**：MSE 服务列表截图、连云上后接口返回截图。

**常见问题**：RDS 白名单要放行本机公网 IP（IP 会变）；Dubbo/Nacos 地址写错前缀。

## 实训五　部署到云：EDAS 主线（4 课时）

**实训目标**：把同一份 jar 部署到云上 3 台 ECS（由 EDAS 托管），前端发布到 ECS-3 的 Nginx，体会 PaaS"少敲命令"的价值。理解 A/B/C 三条路径差异。

**前置条件**：实训三、四完成；已准备 3 台 ECS（2 核 2GB，同 VPC）。

**实训任务**（[EDAS 部署指南](06-edas-deployment.md)，走 **路径 B**）
1. **理解模块结构**：3 个**库模块**（`coffee-common`、两个 `api`）+ 3 个**应用模块**。
2. **制品准备**：`mvn deploy` 把 3 个库发到**云效私有仓库**（构建机没有你的 `~/.m2`）；`mvn package` 打 3 个应用 jar。
3. **建 EDAS 微服务空间**：注册中心类型选 **MSE Nacos**（不用 EDAS 内置）。
4. **建 ECS 集群并导入 3 台 ECS**。
5. **部署 3 个应用**：`userorder`→ECS-1、`expresstrack`→ECS-2、`coffee-app`→ECS-3；JVM 参数 `-DENV=prod -DDB_HOST=... -DDB_USER=... -DDB_PASSWORD=...`（网关不连库，不写 `-DDB_*`；**Agent 自动接管 Dubbo 注册，不写 `-DDUBBO_REGISTRY`**）。
6. **前端部署到 ECS-3 Nginx**（`npm run build` 出 dist → Nginx 托管）。
7. **速览**：路径 A（ECS 自己装 JDK/scp/manage.sh）与路径 C（SAE Serverless，实例数最小 1、虚拟交换机网段加白名单）各看一遍，能说出差异即可。

**预期效果 / 验收标准**
- EDAS"微服务治理→Dubbo→服务查询"能看到接口与提供者 IP。
- 浏览器访问 `http://<ECS-3 公网IP>:8005/actuator/health` 返回 `{"status":"UP"}`；前端可正常下单/查轨迹。

**学生提交物**：EDAS 服务治理截图、健康检查截图、前端云上访问截图。

**常见问题**：必须用 MSE Nacos（内置不支持 Dubbo 3.x）；安全组没放端口；库没 deploy 导致构建找不到 artifact。

## 实训六　CI/CD 流水线自动化（4 课时）

**实训目标**：用云效 Flow 把"`git push` → 构建 → 部署"全自动化，理解流水线源与项目的联动、凭据管理、灰度回滚。

**前置条件**：实训五完成（EDAS 应用已存在）。

**实训任务**（[CI/CD 流水线](07-cicd-pipeline.md)，后端走 **路径 B**）
1. **代码托管 + 源联动**：把代码推到 **Codeup**（或绑定 **GitHub** 源）；理解 **push 到哪个源就触发以那个源为流水线源的流水线**（`git push codeup main` / `git push origin main`）。
2. **全局设置**：建 2 个**服务连接**（阿里云 AK 用 **RAM 子账号**生成、Maven 私库账号）；给子账号授 `AliyunEDASFullAccess`/`AliyunSAEFullAccess`。
3. **通用构建**：Java 构建上传，命令含 `-pl <模块> -am`（`-am` 顺带编译依赖库）；**务必配"构建物上传"**，否则部署找不到 jar。
4. **部署任务**：路径 B 用 **"EDAS 应用发布"**（JVM 参数留空，沿用 EDAS 已有配置）。
5. **前端流水线**：Node.js 模板，dist 发布到 ECS-3 Nginx；`FRONT_API_URL` 按后端路径填。
6. **触发与高级**：监听 `main` 的 Push + 路径过滤；了解灰度（分批发布）与一键回滚。
7. **全链路演示**：改一行代码 `git push`，看流水线全绿、EDAS 出新版本。

**预期效果 / 验收标准**
- 一次 `git push` 后，对应流水线自动跑绿，EDAS/前端出现新版本。

**学生提交物**：流水线运行成功截图、`git push` 触发记录、服务连接配置（密钥打码）。

**常见问题**：构建绿但部署找不到 jar = 没配构建物上传；找不到 artifact = 库没 deploy 或没加 `-am`；主账号 AK 严禁写进流水线。

## 实训七　外网访问与混合云网络（2 课时）

**实训目标**：把"用 IP 访问"升级成"域名 + DNS"；建立一张混合云网络方案地图（看懂即可，只动手域名这一项）。

**前置条件**：实训五完成（前端在 ECS-3）。

**实训任务**（[内外访问](08-cloud-network-access.md)）
1. **动手**：买域名 → （大陆地域）ICP 备案或选海外跳过 → 云解析 DNS **加 A 记录** 把站点域名指向 ECS-3 公网 IP → `nslookup` 验证 → 浏览器用域名打开前端。
2. **看懂**：A 记录（域名→IPv4）与 CNAME（域名→域名）的区别。
3. **开眼界**：VPN 网关 / SAG / 高速通道 / CEN 各记住"一句话"，会用决策树（预算紧流量小→VPN；多分支多云→SAG；要稳大带宽→专线；多 VPC 多地域骨干→CEN）。

**预期效果 / 验收标准**
- `nslookup 你的域名` 指向 ECS-3 公网 IP；浏览器用域名打开前端成功。

**学生提交物**：DNS 解析记录截图、`nslookup` 结果、域名访问前端截图。

**常见问题**：大陆未备案被拦截（教学用海外地域规避）；DNS 生效有缓存，等几分钟。

## 实训八　测试与运维四件套（2 课时）

**实训目标**：上线前压一次、上线后看得见——掌握"云监控管机器、ARMS 管代码、SLS 管日志、PTS 管容量"。

**前置条件**：实训五完成（云上有运行中的应用）。

**实训任务**（[测试与运维](09-testing-and-ops.md)）
1. **PTS 压测**：用 `.jmx` 脚本压 `/order/create` 或健康接口，找到错误率飙升的并发点（容量上限）。
2. **云监控**：给 ECS 配一条"CPU > 80%"告警（钉钉通知）。
3. **ARMS**：给一个 Java 应用加 `-javaagent` 探针，在调用链里定位最慢的一段（如慢 SQL）。
4. **SLS**：在 ECS 装 Logtail 采集 `~/coffee/logs/*.log`，一个搜索框查 3 台机器日志；了解 ActionTrail 投递 SLS"废掉 Root 账号"。

**预期效果 / 验收标准**
- PTS 出一份压测报告（含 RPS/延迟/错误率）；云监控告警规则生效；ARMS 看到一条调用链；SLS 能搜到应用日志。

**学生提交物**：PTS 报告截图、云监控告警截图、ARMS 调用链截图、SLS 查询截图。

**常见问题**：ARMS 探针损耗通常 <3%（延迟极敏感用 eBPF 版）；SLS 用 Logtail 在 ECS 上采集本地日志（走内网，不连 RDS）；PTS 压测前先确认安全组放行了被压接口的端口。

## 实训九　安全加固与等保（2 课时）+ 综合验收

**实训目标**：对照 coffee 项目当前的"5 个洞"做安全加固，理解共享责任模型与 5 条安全底线，知道怎么过等保二级。

**前置条件**：前面实训完成（项目已上云）。

**实训任务**（[安全](10-security.md)）
1. **自检**：列出 coffee 项目当前隐患（RDS/Nacos 白名单 `0.0.0.0/0`、ECS root 登录、coffee-app 裸奔无 WAF、订单表 phone/address 明文）。
2. **加固（任选 2-3 项动手，其余说清思路）**：
   - 收紧 RDS/MSE 白名单为 VPC 内网网段；
   - 给 `coffee.example.com` 接 **WAF**（CNAME 接入）防 SQL 注入/XSS；
   - 订单表 `phone`/`address` 用 **KMS** 加密思路；
   - 三台 ECS 改 **RAM 子账号 + 堡垒机** 代登，ActionTrail 留痕；
   - 把 AccessKey 误推 GitHub 的应急处理（禁用 AK→换新→清历史）。
3. **底线对照**：逐条核对"应用架构 5 条安全底线"。

**预期效果 / 验收标准**
- 完成自检清单；至少落地 2 项加固并截图；能说出过等保二级还差什么（持续监控/渗透/管理文档）。

**学生提交物**：安全自检清单 + 加固截图 + 5 条底线对照表。

### 结项：实训报告（+ 选做答辩）

**最终以实训报告收尾**——把实训各单元的提交物（关键步骤截图 + 预期效果验证 + 遇到的问题与解决）汇总成一份完整实训报告。

- **核心必做（决定基础分）**：实训一～五的提交物齐全——**本地整链路跑通（第 05 章）+ 上云部署成功（第 06 章）**。这是本实训的考核重点，也是大多数学生的最终交付形态。
- **提高加分（选做）**：完成实训六～九中的项并写进报告；学有余力者可对照下方第五节"综合验收清单"逐条打勾，录一段 3–5 分钟演示视频（本地跑通 → 云上访问 → 一次 `git push` 自动发布 → 域名访问 → 一项运维/安全能力）做答辩。
