# ECS 部署作业说明(多 ECS + Workbench 网页 GUI 流程)

## 目标

把三个微服务部署到 **3 台独立 ECS**(一台机器一个微服务),全程使用阿里云 ECS **Workbench 网页终端 + 上传按钮**,不需要本机装 SSH 客户端、不需要 scp 命令。

## ECS 角色规划

| ECS | 跑什么 | 端口 |
|----|------|----|
| ECS-1 | `coffee-userorder-provider-*.jar` | 7001 |
| ECS-2 | `coffee-expresstrack-provider-*.jar` | 8001 |
| ECS-3 | `coffee-app-*.jar` + Nginx 前端 | 8005 + 80 |

---

## 每台 ECS 的目标目录结构

3 台 ECS 都用 **同一份 manage.sh**,`jars/` 里只放本机要跑的那一个 jar:

**ECS-1**:
```
/root/coffee/
├── manage.sh
├── jars/
│   └── coffee-userorder-provider-1.0-SNAPSHOT.jar
└── config/
    └── userorder/
        └── application-dev.yml   ← 已在本地填好 RDS 信息
```

**ECS-2**:
```
/root/coffee/
├── manage.sh
├── jars/
│   └── coffee-expresstrack-provider-1.0-SNAPSHOT.jar
└── config/
    └── expresstrack/
        └── application-dev.yml   ← 已在本地填好 RDS 信息
```

**ECS-3**(不需要 config):
```
/root/coffee/
├── manage.sh
└── jars/
    └── coffee-app-0.0.1-SNAPSHOT.jar
```

---

## 步骤一:本地准备(只做一次)

### A. 本地打包 3 个 jar

在项目根目录:

```bash
cd coffee-userorder/provider && mvn clean package -DskipTests
cd ../../coffee-expresstrack/provider && mvn clean package -DskipTests
cd ../../coffee-app && mvn clean package -DskipTests
```

### B. 用 VSCode/记事本改 3 个配置文件

**`deploy/manage.sh`**:第 20 行附近,把 `NACOS_ADDR` 改成你的 MSE Nacos 内网地址。

**`deploy/config/userorder/application-dev.yml`**:把 `yourDbUser` / `yourDbPassword` / `your-rds-address` 换成你的 RDS 内网地址和账号。

**`deploy/config/expresstrack/application-dev.yml`**:同上(数据库名是 `expresstracktest`)。

---

## 步骤二:打开 3 个 Workbench 标签页

1. 阿里云控制台 → 云服务器 ECS → 实例列表
2. 给每台 ECS 都做一遍:点"远程连接" → "Workbench 远程连接" → 填 root 密码 → 连接
3. 浏览器里现在有 3 个 Workbench 标签页,分别对应 ECS-1 / ECS-2 / ECS-3

---

## 步骤三:每台 ECS 装 JDK + 建目录(粘贴一条命令)

3 个 Workbench 标签页,每个都粘贴同一条命令:

```bash
apt update && apt install -y openjdk-17-jdk && mkdir -p ~/coffee/jars ~/coffee/config && java -version
```

---

## 步骤四:用 Workbench 上传按钮把文件传上去

每台 ECS 的 Workbench 顶部都有 **📤 上传文件** 按钮。

### ECS-1 上传 3 个文件

| 上传 | 本地文件 | 目标路径 |
|----|-------|-------|
| 1 | `deploy/manage.sh` | `/root/coffee/` |
| 2 | `coffee-userorder/provider/target/coffee-userorder-provider-1.0-SNAPSHOT.jar` | `/root/coffee/jars/` |
| 3 | `deploy/config/userorder/application-dev.yml` | `/root/coffee/config/userorder/` |

### ECS-2 上传 3 个文件

| 上传 | 本地文件 | 目标路径 |
|----|-------|-------|
| 1 | `deploy/manage.sh` | `/root/coffee/` |
| 2 | `coffee-expresstrack/provider/target/coffee-expresstrack-provider-1.0-SNAPSHOT.jar` | `/root/coffee/jars/` |
| 3 | `deploy/config/expresstrack/application-dev.yml` | `/root/coffee/config/expresstrack/` |

### ECS-3 只上传 2 个文件

| 上传 | 本地文件 | 目标路径 |
|----|-------|-------|
| 1 | `deploy/manage.sh` | `/root/coffee/` |
| 2 | `coffee-app/target/coffee-app-0.0.1-SNAPSHOT.jar` | `/root/coffee/jars/` |

> 💡 若 Workbench 提示"目标路径不存在",在该 ECS 终端先跑 `mkdir -p ~/coffee/jars ~/coffee/config/userorder`(或 expresstrack),然后重新上传。

---

## 步骤五:每台 ECS 启动服务(粘贴一条命令)

3 个 Workbench 标签页都粘贴同一条:

```bash
cd ~/coffee && chmod +x manage.sh && ./manage.sh start
```

**ECS-1 输出**:启动 userorder,另外两个显示"跳过"
**ECS-2 输出**:启动 expresstrack
**ECS-3 输出**:启动 coffee-app

---

## 步骤六:在网页上配置安全组(鼠标点)

ECS 控制台 → 实例 → 安全组 → 入方向 → 手动添加。

| ECS | 端口 | 用途 |
|-----|-----|------|
| ECS-1 | 7001 | 健康检查(可选) |
| ECS-2 | 8001 | 健康检查(可选) |
| ECS-3 | 8005 | API 网关(浏览器调接口必开) |
| ECS-3 | 80 | Nginx 前端(部署前端时再开) |

## 日常管理

在对应 ECS 的 Workbench 终端里粘贴:

```bash
cd ~/coffee && ./manage.sh status    # 看状态
cd ~/coffee && ./manage.sh logs      # 实时日志(Ctrl+C 退出)
cd ~/coffee && ./manage.sh stop      # 停止
cd ~/coffee && ./manage.sh restart   # 重启
```

换新版本 jar:用 📤 上传按钮覆盖旧 jar → `./manage.sh restart`。
