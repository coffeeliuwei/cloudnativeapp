# ECS 部署作业说明(3 台 ECS + Workbench 网页流程)

把 3 个微服务部署到 3 台独立 ECS,**全程在阿里云网页 Workbench 终端 + 上传按钮** 操作,不用本地装 SSH/scp。

## ECS 角色

| ECS | 跑什么 | 端口 |
|----|------|----|
| ECS-1 | `coffee-userorder-provider.jar` | 7001 |
| ECS-2 | `coffee-expresstrack-provider.jar` | 8001 |
| ECS-3 | `coffee-app.jar` + Nginx 前端 | 8005 + 80 |

每台 ECS 上的目标结构都是 `/root/coffee/` 下放对应的 jar 和配置。

---

## 第 1 步:在本地电脑改好 3 个配置文件

1. 用 VSCode 打开 `cloudnativeapp/deploy/manage.sh`,找到顶部 `NACOS_ADDR="..."`,改成你 MSE Nacos 的**内网地址**,保存
2. 用 VSCode 打开 `cloudnativeapp/deploy/config/userorder/application-dev.yml`,替换 `yourDbUser` / `yourDbPassword` / `your-rds-address` 为你 RDS 实际值,保存
3. 用 VSCode 打开 `cloudnativeapp/deploy/config/expresstrack/application-dev.yml`,同样替换,保存

---

## 第 2 步:本地打包 3 个 jar

在 `cloudnativeapp/` 根目录依次粘贴:

```bash
cd coffee-userorder/provider && mvn clean package -DskipTests
```
```bash
cd ../../coffee-expresstrack/provider && mvn clean package -DskipTests
```
```bash
cd ../../coffee-app && mvn clean package -DskipTests
```

每条最后看到 `BUILD SUCCESS` 即成功。

---

## 第 3 步:打开 3 个 Workbench 标签页

4. 浏览器打开 [阿里云 ECS 控制台](https://ecs.console.aliyun.com)
5. 找到 **ECS-1** → 最右边点 **"远程连接"** → 选 **"Workbench 远程连接"** → 输入 root 密码 → 连接
6. 浏览器出现一个黑底白字终端,这就是 ECS-1 的命令行
7. 对 **ECS-2** 和 **ECS-3** 各开一个新标签页做同样操作

---

## 第 4 步:每台 ECS 装 JDK + 建目录

8. 切到任一 Workbench 标签页,粘贴并回车:
   ```bash
   apt update && apt install -y openjdk-17-jdk && mkdir -p ~/coffee/jars ~/coffee/config && java -version
   ```
9. 最后一行看到 `openjdk version "17.0.x"` 即成功
10. **对另外 2 台 ECS** 重复一次

---

## 第 5 步:用 Workbench 上传按钮把文件传上去

每台 Workbench 顶部有 **📤 上传文件** 按钮。

### ECS-1(订单)上传 3 个文件

| 第几次 | 本地文件 | ECS 目标路径 |
|------|---------|------------|
| 1 | `deploy/manage.sh` | `/root/coffee/` |
| 2 | `coffee-userorder/provider/target/coffee-userorder-provider-1.0-SNAPSHOT.jar` | `/root/coffee/jars/` |
| 3 | `deploy/config/userorder/application-dev.yml` | `/root/coffee/config/userorder/` |

### ECS-2(快递)上传 3 个文件

| 第几次 | 本地文件 | ECS 目标路径 |
|------|---------|------------|
| 1 | `deploy/manage.sh` | `/root/coffee/` |
| 2 | `coffee-expresstrack/provider/target/coffee-expresstrack-provider-1.0-SNAPSHOT.jar` | `/root/coffee/jars/` |
| 3 | `deploy/config/expresstrack/application-dev.yml` | `/root/coffee/config/expresstrack/` |

### ECS-3(网关)只上传 2 个文件

| 第几次 | 本地文件 | ECS 目标路径 |
|------|---------|------------|
| 1 | `deploy/manage.sh` | `/root/coffee/` |
| 2 | `coffee-app/target/coffee-app-0.0.1-SNAPSHOT.jar` | `/root/coffee/jars/` |

> 如果上传时提示"目标路径不存在",先在该 ECS 终端跑一句 `mkdir -p ~/coffee/jars ~/coffee/config/userorder`(或 expresstrack),再重传。

---

## 第 6 步:每台 ECS 启动服务

11. 3 台 Workbench 都粘贴**同一条命令**:
    ```bash
    cd ~/coffee && chmod +x manage.sh && ./manage.sh start
    ```
12. 等约 30 秒,看到对应服务 `[✓] 启动成功` 即可
13. **顺序建议**:先 ECS-1 / ECS-2(provider),最后 ECS-3(consumer)

> 输出里另外两个服务显示"跳过"是正常的——脚本检测到本机没那个 jar 就跳过,这样一份脚本通用 3 台。

---

## 第 7 步:在网页 GUI 放开端口

ECS 控制台 → 找对应 ECS → 安全组 → 入方向 → 手动添加(协议 TCP,授权对象 `0.0.0.0/0`):

| ECS | 端口 |
|-----|------|
| ECS-1 | 7001 |
| ECS-2 | 8001 |
| ECS-3 | 8005, 80 |

---

---

## 日常管理

回对应 ECS 的 Workbench 终端粘命令:

```bash
cd ~/coffee && ./manage.sh status    # 看本机服务状态
cd ~/coffee && ./manage.sh logs      # 实时看日志(Ctrl+C 退出)
cd ~/coffee && ./manage.sh stop      # 停本机服务
cd ~/coffee && ./manage.sh restart   # 重启
```

**换新版本 jar**:📤 上传新 jar 覆盖旧的 → `cd ~/coffee && ./manage.sh restart`
