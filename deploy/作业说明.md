# ECS 部署作业说明（多 ECS 微服务部署）

## 目标

将三个微服务部署到 **3 台独立的 ECS** 上，遵循"一台机器一个微服务"的标准微服务架构。

## ECS 角色规划

| ECS 编号 | 角色 | 部署内容 | 端口 |
|---------|------|---------|------|
| ECS-1 | 订单服务 | `coffee-userorder-provider-*.jar` | 7001 |
| ECS-2 | 快递服务 | `coffee-expresstrack-provider-*.jar` | 8001 |
| ECS-3 | 网关 + 前端 | `coffee-app-*.jar` + Nginx 前端 | 8005 + 80 |

> ECS-3 同时承担 API 网关和前端入口的角色。生产环境通常前端走 CDN 或独立 ECS，教学环境合并以省成本。

---

## 每台 ECS 的目录结构

每台 ECS 都用 **同一份 manage.sh**，但 `jars/` 下只放本机要跑的那个 jar。脚本会自动识别本机有哪个 jar，只启动对应的服务。

**ECS-1（订单服务）**：
```
/opt/coffee/
├── jars/
│   └── coffee-userorder-provider-1.0-SNAPSHOT.jar
├── config/
│   └── userorder/
│       └── application-dev.yml     ← 填你的 RDS 信息
├── logs/
└── manage.sh
```

**ECS-2（快递服务）**：
```
/opt/coffee/
├── jars/
│   └── coffee-expresstrack-provider-1.0-SNAPSHOT.jar
├── config/
│   └── expresstrack/
│       └── application-dev.yml     ← 填你的 RDS 信息
├── logs/
└── manage.sh
```

**ECS-3（网关 + 前端）**：
```
/opt/coffee/
├── jars/
│   └── coffee-app-0.0.1-SNAPSHOT.jar
├── logs/
└── manage.sh
（coffee-app 不连数据库，不需要 config/ 目录）
```

---

## 步骤一：本地打包（只做一次）

在本地项目根目录分别执行：

```bash
cd coffee-userorder/provider
mvn clean package -DskipTests

cd ../../coffee-expresstrack/provider
mvn clean package -DskipTests

cd ../../coffee-app
mvn clean package -DskipTests
```

打包产物在各模块的 `target/` 目录。

---

## 步骤二：每台 ECS 安装 JDK 17

**3 台 ECS 都要做一次**，SSH 登录后执行：

```bash
# Ubuntu
sudo apt update
sudo apt install -y openjdk-17-jdk

# Alibaba Cloud Linux
sudo dnf install -y java-17-openjdk-devel

java -version   # 确认输出包含 "17"
```

---

## 步骤三：上传文件到各 ECS

把 `<ECS-1_IP>`、`<ECS-2_IP>`、`<ECS-3_IP>` 替换为对应公网 IP。

**ECS-1：上传 manage.sh、userorder 配置、userorder jar**：
```bash
ssh root@<ECS-1_IP> "mkdir -p /opt/coffee/jars"
scp deploy/manage.sh root@<ECS-1_IP>:/opt/coffee/
scp -r deploy/config/userorder root@<ECS-1_IP>:/opt/coffee/config/
scp coffee-userorder/provider/target/coffee-userorder-provider-1.0-SNAPSHOT.jar \
    root@<ECS-1_IP>:/opt/coffee/jars/
```

**ECS-2：上传 manage.sh、expresstrack 配置、expresstrack jar**：
```bash
ssh root@<ECS-2_IP> "mkdir -p /opt/coffee/jars"
scp deploy/manage.sh root@<ECS-2_IP>:/opt/coffee/
scp -r deploy/config/expresstrack root@<ECS-2_IP>:/opt/coffee/config/
scp coffee-expresstrack/provider/target/coffee-expresstrack-provider-1.0-SNAPSHOT.jar \
    root@<ECS-2_IP>:/opt/coffee/jars/
```

**ECS-3：只上传 manage.sh 和 coffee-app jar（不需要 config）**：
```bash
ssh root@<ECS-3_IP> "mkdir -p /opt/coffee/jars"
scp deploy/manage.sh root@<ECS-3_IP>:/opt/coffee/
scp coffee-app/target/coffee-app-0.0.1-SNAPSHOT.jar \
    root@<ECS-3_IP>:/opt/coffee/jars/
```

---

## 步骤四：每台 ECS 配置 Nacos 地址

3 台 ECS 都要做：SSH 登录后编辑 `/opt/coffee/manage.sh`，第 20 行附近：

```bash
nano /opt/coffee/manage.sh
# 把 NACOS_ADDR 改成你自己的 MSE Nacos 内网地址
NACOS_ADDR="mse-xxxxxxxx-p.nacos-ans.mse.aliyuncs.com:8848"
```

3 台都填同一个 Nacos 地址（同一个注册中心）。

---

## 步骤五：填写数据库配置（只在 ECS-1 和 ECS-2）

**ECS-1**：
```bash
nano /opt/coffee/config/userorder/application-dev.yml
# 把 yourDbUser、yourDbPassword、your-rds-address 换成实际值（RDS 内网地址）
```

**ECS-2**：
```bash
nano /opt/coffee/config/expresstrack/application-dev.yml
```

**ECS-3 不需要**：coffee-app 不连数据库，没有 config 目录。

> 注意：ECS 和 RDS 必须在同一 VPC，且 RDS 白名单要加 3 台 ECS 的私网 IP。

---

## 步骤六：在每台 ECS 启动服务

3 台 ECS 上分别执行（同一个命令）：

```bash
chmod +x /opt/coffee/manage.sh
/opt/coffee/manage.sh start
```

**ECS-1 上你会看到**：
```
>>> 启动本机部署的服务
    本机检测到 1 个 jar，开始按依赖顺序启动...
  [✓] userorder: 启动成功 (PID=xxxx, 端口=7001)
  [-] expresstrack: 跳过（本机未部署此服务）
  [-] app: 跳过（本机未部署此服务）
```

**ECS-2 上**：只启动 expresstrack，其他跳过。
**ECS-3 上**：只启动 coffee-app，其他跳过。

---

## 启动顺序建议

理论上：先启 ECS-1 和 ECS-2（provider），再启 ECS-3（consumer）。
实际上：顺序反了也能启动，coffee-app 启动时如果暂时找不到 provider，Dubbo 会自动重试。

---

## 日常管理命令

每台 ECS 各自管理：
```bash
./manage.sh status    # 查看本机服务运行状态
./manage.sh logs      # 实时查看本机日志（Ctrl+C 退出）
./manage.sh stop      # 停止本机服务
./manage.sh restart   # 重启本机服务
```

## 验证

| 验证内容 | 地址 |
|---------|------|
| ECS-1 订单服务健康 | `http://<ECS-1_IP>:7001/actuator/health` |
| ECS-2 快递服务健康 | `http://<ECS-2_IP>:8001/actuator/health` |
| ECS-3 网关健康 | `http://<ECS-3_IP>:8005/actuator/health` |
| MSE Nacos 控制台 → 服务列表 | 应看到 3 个服务都已注册 |

返回 `{"status":"UP"}` 表示正常。

---

## 端口与安全组速查

| ECS | 入方向开放端口 | 用途 |
|-----|--------------|------|
| ECS-1 | 22, 7001 | SSH、健康检查 |
| ECS-2 | 22, 8001 | SSH、健康检查 |
| ECS-3 | 22, 80, 8005 | SSH、Nginx 前端、API 网关（前端浏览器调用） |

所有 ECS 都要开放 **出方向** 到 MSE Nacos 和 RDS 的端口（默认已通）。
