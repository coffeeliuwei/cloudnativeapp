# ECS 部署作业说明

## 目标

将三个微服务打包后上传到阿里云 ECS，使用管理脚本一键启动。

---

## 目录结构

上传到 ECS 后，目录应如下所示：

```
/opt/coffee/
├── jars/
│   ├── coffee-userorder-provider-1.0-SNAPSHOT.jar
│   ├── coffee-expresstrack-provider-1.0-SNAPSHOT.jar
│   └── coffee-app-0.0.1-SNAPSHOT.jar
├── config/
│   ├── userorder/
│   │   └── application-dev.yml      ← 填写你的数据库配置
│   └── expresstrack/
│       └── application-dev.yml      ← 填写你的数据库配置
├── logs/                            ← 自动创建，存放运行日志
└── manage.sh                        ← 管理脚本
```

---

## 步骤一：本地打包

在本地项目根目录分别执行：

```bash
# 打包 userorder
cd coffee-userorder/provider
mvn clean package -DskipTests

# 打包 expresstrack
cd ../../coffee-expresstrack/provider
mvn clean package -DskipTests

# 打包 coffee-app
cd ../../coffee-app
mvn clean package -DskipTests
```

生成的 JAR 在各模块的 `target/` 目录下。

---

## 步骤二：ECS 安装 JDK 17

SSH 登录 ECS 后执行：

```bash
sudo apt update
sudo apt install -y openjdk-17-jdk
java -version   # 确认输出包含 "17"
```

---

## 步骤三：上传文件到 ECS

```bash
# 在本地执行（将 <ECS_IP> 替换为你的 ECS 公网 IP）
scp coffee-userorder/provider/target/coffee-userorder-provider-1.0-SNAPSHOT.jar root@<ECS_IP>:/opt/coffee/jars/
scp coffee-expresstrack/provider/target/coffee-expresstrack-provider-1.0-SNAPSHOT.jar root@<ECS_IP>:/opt/coffee/jars/
scp coffee-app/target/coffee-app-0.0.1-SNAPSHOT.jar root@<ECS_IP>:/opt/coffee/jars/

# 上传脚本和配置
scp -r deploy/manage.sh deploy/config root@<ECS_IP>:/opt/coffee/
```

---

## 步骤四：配置 Nacos 地址和数据库

**1. 修改 manage.sh 中的 Nacos 地址**

SSH 登录 ECS，编辑脚本：
```bash
nano /opt/coffee/manage.sh
```
将第 14 行的 `NACOS_ADDR` 替换为你自己的 MSE Nacos 公网地址：
```
NACOS_ADDR="mse-xxxxxxxx-p.nacos-ans.mse.aliyuncs.com:8848"
```

**2. 填写数据库配置**

```bash
nano /opt/coffee/config/userorder/application-dev.yml
nano /opt/coffee/config/expresstrack/application-dev.yml
```
将 `yourDbUser`、`yourDbPassword`、`your-rds-address` 替换为实际值。

> 注意：ECS 和 RDS 在同一 VPC 内，使用 RDS **内网地址**即可。

---

## 步骤五：启动服务

```bash
chmod +x /opt/coffee/manage.sh
/opt/coffee/manage.sh start
```

---

## 日常管理命令

```bash
./manage.sh status    # 查看三个服务是否正常运行
./manage.sh logs      # 实时查看日志（Ctrl+C 退出）
./manage.sh stop      # 停止所有服务
./manage.sh restart   # 重启所有服务
```

---

## 验证

服务启动后，在浏览器访问（将 IP 替换为你的 ECS 公网 IP）：

| 服务 | 验证地址 |
|------|---------|
| coffee-app（API 网关） | `http://<ECS_IP>:8005/actuator/health` |
| coffee-userorder | `http://<ECS_IP>:7001/actuator/health` |
| coffee-expresstrack | `http://<ECS_IP>:8001/actuator/health` |

返回 `{"status":"UP"}` 表示正常。

---

## 端口说明

| 服务 | HTTP 端口 | 说明 |
|------|----------|------|
| coffee-userorder | 7001 | 用户订单服务 |
| coffee-expresstrack | 8001 | 快递轨迹服务 |
| coffee-app | 8005 | API 网关，前端连接此端口 |
